RPA-001 | USER REGISTRATION & ACCOUNT MANAGEMENT
────────────────────────────────────────────────────────────────────────────────
Purpose            : Create and maintain user accounts; enable personalised
                     access to platform features; verify identity; manage
                     account preferences and language settings.

Data Principal     : Registered end-users (general public, age ≥ 18 years)

Personal Data Collected:
  - Full name
  - Email address
  - Mobile phone number
  - Password (stored as bcrypt hash; plaintext never retained)
  - Profile photo (optional)
  - Preferred language(s): Tamil / Hindi / Telugu / English
  - State / district (optional, for regional content routing)
  - Date of birth (age verification)
  - Gender (optional)
  - Device identifiers (Device ID, Advertising ID, OS, App version)

Sensitive Personal Data: None

Legal Basis (DPDP Act 2023):
  - Section 4 read with Section 6 – Consent provided during signup via
    freely given, specific, informed, and unambiguous opt-in

Legal Basis (supplementary, GDPR Art 6 for EU-resident users):
  - Art. 6(1)(b) – Performance of contract

Data Processors / Third Parties:
  - AWS (Amazon Web Services) – cloud hosting, Mumbai region (primary)
  - Twilio India Pvt. Ltd. – OTP / SMS verification
  - Firebase (Google LLC) – mobile authentication tokens

Cross-border Transfers:
  - Singapore (AWS ap-southeast-1, failover only) – Standard Contractual
    Clauses in place

Retention:
  - Active accounts: Duration of account + 3 years post-deletion request
  - Deleted accounts: PII purged within 30 days of verified deletion request;
    anonymised aggregate statistics retained indefinitely
  - Legal hold override: extended as required per applicable law

Security Measures:
  - Passwords hashed with bcrypt (cost factor 12)
  - Data in transit: TLS 1.3
  - Data at rest: AES-256 encryption on RDS instances (AWS Mumbai)
  - Multi-factor authentication available for all users
  - Role-based access control for internal staff

Automated Decision Making: None

RPA-002 | CONTENT PERSONALISATION & RECOMMENDATION ENGINE
────────────────────────────────────────────────────────────────────────────────
Purpose            : Curate the news feed and video playlist based on user
                     interests, viewing history, language preferences, and
                     geographic signals; serve region-specific Tamil Nadu,
                     Andhra Pradesh, Telangana, and North India editions.

Data Principal     : Registered users; anonymous visitors (limited profiling)

Personal Data Processed:
  - User ID (pseudonymous)
  - Content interaction events (video plays, article opens, shares, saves,
    reactions, skip events, watch time per clip)
  - Search queries entered on platform
  - Language preference
  - Geographic region (state/district level derived from IP; pin code if
    user provides)
  - Device type (mobile / desktop / tablet)
  - Time-of-day usage patterns

Sensitive Personal Data:
  - Inferred political interest signals derived from content categories viewed
    – treated as sensitive; NOT shared with third parties; used only for
    on-platform personalisation with appropriate access controls

Legal Basis (DPDP Act 2023):
  - Section 6 – Consent (users agree to personalisation during onboarding;
    toggleable in Privacy Settings under "My Preferences")
  - Legitimate use where consent not withdrawn

Data Processors:
  - AWS SageMaker – ML model training and inference (Mumbai region)
  - Apache Kafka (self-hosted on AWS EC2) – event streaming pipeline

Cross-border Transfers: None (all ML workloads in AWS Mumbai ap-south-1)

Retention:
  - Raw interaction logs: 13 months (rolling)
  - Aggregated behavioural profiles: 36 months or until account deletion
  - ML model weights: retained until superseded by newer model version

Security Measures:
  - Pseudonymisation: internal user IDs replace PII in all ML pipelines
  - No re-identification without dual-authorisation workflow
  - Access restricted to Data Science and Platform Engineering teams

Automated Decision Making:
  - Fully automated recommendation ranking; no decisions with legal or
    similarly significant effect; no human review step required by policy

RPA-017 | JOURNALIST TIP-OFF / SECURE SOURCE CHANNEL
────────────────────────────────────────────────────────────────────────────────
Purpose            : Provide a confidential, encrypted channel for
                     whistleblowers and sources to submit information to
                     Thirai24's investigative journalists, with strong source
                     protection commitments in line with journalistic ethics.

Data Principal     : Individuals submitting tips (identities strongly protected)

Personal Data Processed:
  - Content of tip / document (may contain PII of third parties)
  - No IP addresses logged (SecureDrop over Tor architecture)
  - Ephemeral codename assigned to source by SecureDrop system
  - No email or phone number collected at any point

Sensitive Personal Data:
  - Potentially highly sensitive depending on tip content; treated with
    maximum protection classification

Legal Basis (DPDP Act 2023):
  - Section 6 – Consent (source initiates contact voluntarily)
  - Journalistic purpose exemption under DPDP Act Rules and Programme for
    the Protection of Journalistic Sources

Data Processors: None (SecureDrop is self-hosted; no third-party processors)

Cross-border Transfers: None

Retention:
  - Tips reviewed by assigned investigative journalist; deleted from server
    within 90 days of review completion unless active investigation requires
    retention; maximum retention: 1 year in any circumstance

Security Measures:
  - SecureDrop instance air-gapped from production network (Chennai server
    room, restricted physical access)
  - End-to-end encryption; Tor network hides source IP
  - Only two designated investigative journalists have server access
  - Annual third-party security audit of SecureDrop instance

Automated Decision Making: None
