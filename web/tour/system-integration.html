5 RBAC Permission Scope Mapping

This section maps the Consent Management System (CMS) functions to the finalized Permission Scopes for implementation in the Auth service.

1. Write Scope

Grant this scope to Data Principals and Applications capturing new intent or updating interaction states.

Function (_func) Description
record_consentCaptures primary notice acceptance.
record_parent_consentRegisters verifiable parental proof for children (Section 9).
link_userAssociates anonymous identifiers with authenticated user accounts.
withdraw_consentModifies active consent records to revoke specific rights.
submit_grievanceInitiates the rights exercise flow (Section 13).
mark_notification_readManagement of the dashboard notification center.

2. Read Scope

Grant this scope to Dashboards, Processors, and Compliance auditing modules for information retrieval.

Function (_func) Description
get_active_consentRetrieves the current valid consent state for a principal.
list_consent_historyReturns the temporal trail of all consent events.
get_consent_record_detailsProvides evidentiary details (Timestamp, IP, Notice Version).
validate_consentReal-time verification of authorization for processors.
get_grievanceProvides status updates on registered complaints.
list_user_grievancesLists historical complaints for a specific principal.
get_policyRetrieves a specific version of a privacy policy for archival view.
get_active_policyFetches the current live policy for notice rendering.
list_notificationsView system alerts regarding purge milestones or notice changes.

3. Purge Scope

Grant this scope to Data Principals (initiation) and DPOs (fulfillment) to manage the data end-of-life lifecycle.

Function (_func) Description
erasure_requestThe Data Principal confirms the intent to delete all personal data.
list_purge_requestsOversee the queue of pending data deletions across fiduciaries.
update_purge_statusManual/Job override to confirm technical data destruction.
